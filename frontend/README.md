# Provenance AI — AI Code Detector

A forensic web application that detects whether a piece of code was written by a human or generated by an AI. It uses a multi-layered analysis engine combining heuristic pattern matching, NLP transformer models, structural code analysis, and multi-model prompt comparison.

## Architecture

```
code detector/
├── backend/          # FastAPI (Python) — analysis engine & API
│   ├── main.py           # API endpoints
│   ├── heuristics.py     # Rule-based pattern detection
│   ├── nlp.py            # NLP/Transformer analysis (CodeBERT)
│   ├── similarity.py     # Multi-layer code similarity engine
│   ├── prompt_engine.py  # Multi-provider LLM code generation
│   ├── ml_weighting.py   # ML-based score combination
│   ├── database.py       # Supabase client
│   └── requirements.txt
├── frontend/         # React + TypeScript + Vite — UI
│   └── src/App.tsx
└── docker-compose.yml
```

## Prerequisites

- **Python 3.10+**
- **Node.js 18+**
- **Docker & Docker Compose** (for containerized deployment)

## Running Locally (Development)

### 1. Backend

```bash
cd backend

# Create and activate a virtual environment
python -m venv venv
# Windows:
venv\Scripts\activate
# macOS/Linux:
# source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Configure environment variables
# Copy the example and fill in your API keys
cp .env.example .env
# Edit .env with your keys (see Configuration section below)

# Start the server
uvicorn main:app --reload --port 8000
```

The API will be available at `http://127.0.0.1:8000`.

### 2. Frontend

```bash
cd frontend

# Install dependencies
npm install

# Start the dev server
npm run dev
```

The UI will be available at `http://localhost:5173`.

> **Note:** In development mode, the frontend automatically connects to the backend at `http://127.0.0.1:8000`. No extra configuration is needed.

## Running with Docker (Production)

From the root of the project (`code detector/`):

```bash
# Build and start both services
docker-compose up -d --build
```

| Service  | URL                    |
|----------|------------------------|
| Frontend | `http://localhost:3000` |
| Backend  | `http://localhost:8000` |

To stop:

```bash
docker-compose down
```

To set a custom backend URL for the frontend (e.g., when deploying to separate servers):

```bash
VITE_API_URL=https://your-api.example.com docker-compose up -d --build
```

## Configuration

Create a `backend/.env` file with the following variables. You only need to configure **one** LLM provider — the rest are optional.

### Required

| Variable           | Description                          |
|--------------------|--------------------------------------|
| `SUPABASE_URL`     | Your Supabase project URL            |
| `SUPABASE_KEY`     | Your Supabase anon/public key        |

### LLM Provider (choose one)

Set `PROMPT_ENGINE_PROVIDER` to one of: `groq`, `together`, `huggingface`, `gemini`, `sambanova`, `cerebras`, or `ollama`.

| Variable               | Provider      | Where to get a key                        |
|------------------------|---------------|-------------------------------------------|
| `GROQ_API_KEY`         | Groq          | https://console.groq.com                  |
| `TOGETHER_API_KEY`     | Together AI   | https://api.together.ai                   |
| `HUGGINGFACE_API_KEY`  | Hugging Face  | https://huggingface.co/settings/tokens    |
| `GEMINI_API_KEY`       | Google Gemini | https://aistudio.google.com/app/apikey    |
| `SAMBANOVA_API_KEY`    | SambaNova     | https://cloud.sambanova.ai                |
| `CEREBRAS_API_KEY`     | Cerebras      | https://cloud.cerebras.ai                 |

> **Ollama** requires no API key — just have it running locally at `http://localhost:11434`.

### Optional overrides

| Variable           | Default                             |
|--------------------|-------------------------------------|
| `GROQ_MODEL`       | `llama-3.3-70b-versatile`          |
| `TOGETHER_MODEL`   | `Qwen/Qwen2.5-Coder-7B-Instruct`  |
| `HF_MODEL`         | `Qwen/Qwen2.5-Coder-1.5B-Instruct`|
| `GEMINI_MODEL`     | `gemini-2.0-flash`                 |
| `SAMBANOVA_MODEL`  | `Meta-Llama-3.3-70B-Instruct`      |
| `CEREBRAS_MODEL`   | `llama3.1-70b`                     |

## API Endpoints

| Method | Endpoint            | Description                                      |
|--------|---------------------|--------------------------------------------------|
| POST   | `/analyze`          | Analyze code for AI-generation probability       |
| POST   | `/analyze/prompt`   | Analyze code + compare against LLM-generated baseline |
| GET    | `/history`          | Fetch recent scan history from Supabase          |
| GET    | `/health`           | Health check                                     |
| GET    | `/provider/status`  | Current LLM provider info                        |
| GET    | `/ollama/status`    | Local Ollama availability                        |
